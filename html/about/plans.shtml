About UCSDMap

Plans

Many parts of the application are not yet complete. This is a very brief outline of what we still plan to do. (This will all probably get done between the end of Summer Session I and the beginning of Fall 2005.) Crossed-off items have been completed.

  • A better base map. The current map is just the visitor map PDF. (It even still has the key in the upper right-hand corner!) We plan on using the colored visitor map, such as is used by maps.ucsd.edu.
  • Complete path data. The current path data is for demonstration only and is in no way complete. Once we obtain a final base map and create a scaled base image for use in the editor, we'll enter in the complete path data.
  • Complete building codes. The application supports associating a building code with each location (i.e., "PCYNH" with "Pepper Canyon Hall"), but they have not been entered.
  • Google Maps-style scrolling. Who can resist?
  • Other interface improvements. We'll get rid of some of the existing rough edges in the interface to make it easier and quicker to use.

...plus anything else that comes up.

Extensibility

These features are not specifically planned, but they are listed to give an idea of how this project could expand:

  • Building maps. Click on an individual building name to open a sub-map (complete with all the features of the existing map -- locations, path finding, etc) of the inside of that building.
  • Maps of other locations. With the editor, it is easy to add path data to any map.
  • Integration with other applications. Each location may have an associated building code, so it is easy for external applications that know building codes to build URLs for the map. I.e., to show the path between CENTR and PCYNH, you simply add the following to the map URL: "?from=CENTR&to=PCYNH".
