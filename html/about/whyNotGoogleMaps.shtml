About UCSDMap

Why Not Google Maps?

The Google Maps API, which allows developers to use Google Maps to power their own applications, may seem to eliminate the need for a specialized application like UCSDMap. However, largely because it focuses on city roads and driving directions, and we do not believe it suits the needs of UCSD.

Advantages of UCSDMap

  • Access to detailed maps. UCSD has detailed maps of campus and high-resolution satellite images, which UCSDMap can readily use. Google Maps is limited to Google's map, which is much less detailed, and satellite images, which are lower-resolution.
  • Flexibility. The editor we created to enter path data can be used to enter path data for any base map, not just the UCSD campus. If UCSD ever needs internal building maps, or other maps that Google would not have, they can readily be created and used.
  • Visual consistency. UCSDMap can (and in fact plans to) use the maps that UCSD already has displayed on kiosks around campus. Google Maps uses only the maps Google has created.
  • Future reliability. UCSDMap does not rely on the goodwill of an external company -- in the Google Maps API Terms of Use (section 1.5), Google explicitly reserves the right to include advertisements.
  • General extensibility. Should some unforeseen circumstance require the map to possess a new feature, UCSD will have access to the full code base of UCSDMap, so any necessary changes can be made. See the Extensibility section of the plans page.

Advantages of Google Maps

...and how we can match (or beat) them.

  • Hybrid view. This is the satellite image overlaid with road maps.
    • Because UCSD has both satellite images and detailed maps of campus, it would not be difficult to create a "hybrid" map like Google's by overlaying the road layer of one of the visitor maps on the satellite image. This would have the advantage of optionally including foot paths, and anything else that appears on UCSD's existing maps, instead of just public roads.
    • In a nutshell, it is possible to create a hybrid map that is more flexible than Google's.
  • Live scrolling/dragging/zooming.
    • This is a planned feature of UCSDMap. We plan on implementing it, at the latest, before the beginning of Fall quarter 2005.
  • Looks.
    • The appearance of UCSDMap is template-based, and can be changed easily. Multiple looks are possible simultaneously (such as a standard view and a printing view).
